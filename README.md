# Humanitarian Needs & Response Dashboard for Ukraine

This is my first portfolio project in policy-oriented data analytics.

The goal of this project is to analyze humanitarian needs, displacement pressure, and response coverage across Ukrainian regions. I want to build a simple but useful analytical workflow that connects data analysis with real decision-making needs in the humanitarian and public sector.

## Research question

Which Ukrainian regions show the highest combination of humanitarian need, displacement pressure, and security risk?

## Why this project matters

Humanitarian and public-sector decisions often depend on messy, incomplete, and fast-changing data. This project is designed to show how open data can be cleaned, structured, analyzed, and turned into a dashboard that helps identify possible regional gaps in assistance.

## Tools I plan to use

- **R** for the main analysis, descriptive statistics, visualizations, and simple statistical models
- **SQL / BigQuery** for preparing and aggregating structured data
- **Python** for cleaning messy files and automating small data tasks
- **Power BI** for building an interactive dashboard
- **GIS / mapping** for regional visualization

## Planned data sources

The project will use open data related to:

- Ukrainian administrative regions
- internally displaced persons and returnees
- humanitarian needs indicators
- conflict or security-related events
- response coverage where available

## Planned workflow

1. Define the policy question and indicators
2. Collect open humanitarian and public-sector data
3. Clean and standardize regional names and variables
4. Build a region-level analytical dataset
5. Explore the data in R
6. Create simple indicators and a prototype pressure score
7. Build a Power BI dashboard
8. Write a short analytical brief with findings and limitations

## Expected outputs

- Clean region-level dataset
- R scripts for analysis
- SQL query examples
- Python cleaning script
- Power BI dashboard screenshots
- Short policy brief
- Clear README and methodology notes
